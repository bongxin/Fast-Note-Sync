#### 【活动者】
品控部

#### 【需求描述】

在 Web 端对【消毒登记（小程序）】提交的清洗消毒登记记录进行审核，支持按条件筛选待审及已审数据，查看登记详情与现场照片等佐证材料，完成审核通过操作，并按固定模板导出清洗消毒记录表。

#### 【业务规则】

##### 标题
消毒登记审核

##### 筛选条件

|序号|字段名称|数据类型|必填|默认值|说明/交互规范|
|:--|:--|:--|:--|:--|:--|
|1|**日期范围**|日期范围选择器|否|近 7 天|选填。  <br>支持选择起止日期，格式 YYYY-MM-DD。  <br>默认展示近 7 天内的记录。|
|2|**消毒药物名称**|下拉选择框|否|全部|选填。  <br>选项取自系统消毒药物主数据，含“全部”。|
|3|**审核状态**|下拉选择框|否|待审核|**选项如下：**  <br>- 全部<br>- 待审核<br>- 已通过|
|4|**清洗人员**|文本输入框|否|无|选填。  <br>支持按清洗人员姓名模糊查询。|

##### 列表字段

Web 端以表格形式展示，支持分页。数据来源为【消毒登记（小程序）】已保存的登记记录。

|序号|字段名称|数据类型|说明/交互规范|
|:--|:--|:--|:--|
|1|**清洗日期**|日期|展示清洗日期，格式 YYYY-MM-DD。|
|2|**清洗时间**|时间|展示清洗时间，格式 HH:mm:ss。|
|3|**消毒药物名称**|文本|展示所选消毒药物。|
|4|**浓度比率**|文本|展示浓度比率，如 1:200。|
|5|**消毒方案**|文本|展示消毒方案，默认“喷洗”。|
|6|**清洗人员**|文本|展示提交登记人员姓名。|
|7|**提交时间**|日期时间|展示记录提交时间，格式 YYYY-MM-DD HH:mm。|
|8|**审核状态**|标签|展示“待审核”或“已通过”。  <br>“待审核”以醒目颜色标识。|
|9|**审核人**|文本|已审核时展示审核人姓名；待审核时显示“—”。|
|10|**审核时间**|日期时间|已审核时展示审核时间，格式 YYYY-MM-DD HH:mm；待审核时显示“—”。|
|11|**操作**|操作列|待审核记录显示“审核”链接；已审核记录显示“查看”链接。|

_列表默认按提交时间倒序排列。_

##### 审核项

对待审核记录，审核人在详情抽屉底部填写并提交：

|序号|字段名称|数据类型|必填|不可编辑|默认值|说明/交互规范|
|:--|:--|:--|:--|:--|:--|:--|
|1|**审核人**|文本|是|是|当前登录用户|自动获取当前登录的品控部人员姓名，不可编辑。|
|2|**审核人签名**|签名图片|是|是|【我的】-【签名管理】的签名（如有）|自动获取当前审核人在【签名管理】中的签名图片；无签名时提示先维护签名。  <br>不可手动修改。|
|3|**审核意见**|多行文本|否|否|无|选填。  <br>占位符提示：“请输入审核意见”。|

|1|**查询**|位于筛选区，点击后按当前筛选条件刷新列表。|
|2|**重置**|位于筛选区，点击后恢复筛选条件为默认值并刷新列表。|

- 筛选范围内无登记数据的日期不生成空行。
- 超出单页容量时自动分页，页脚标注“第 N 页”。

**文件命名**

#### 【操作流程】

1. **进入页面**：审核人员登录 Web 管理系统，进入【屠宰监管】-【消毒登记审核】，系统默认加载近 7 天内、状态为“待审核”的记录列表。
2. **筛选查询**：按需设置日期范围、消毒药物名称、审核状态、清洗人员，点击“查询”刷新列表；点击“重置”恢复默认筛选。
3. **浏览列表**：在表格中查看各条登记的清洗日期时间、消毒药物、清洗人员及审核状态；优先处理“待审核”记录。
4. **审核处理**：点击待审核记录的“审核”，在抽屉中核对登记信息、现场照片及清洗人员签名；可填写审核意见，点击“通过”完成审核。
5. **查看已审记录**：点击已审核记录的“查看”，在抽屉中浏览登记信息与审核结果，不可再次修改审核结论。
6. **导出记录**：设置筛选条件后点击“导出”，系统生成并下载符合模板格式的 Excel 清洗消毒记录表。
