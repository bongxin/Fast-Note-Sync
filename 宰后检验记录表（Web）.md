#### 【活动者】
品控部

#### 【需求描述】

在 Web 端按日展示《宰后检验记录表》，自动汇聚工控机屠宰数量、死猪登记、病害肉登记及小程序宰后检验登记数据，支持复核审核，形成完整宰后检验台账。

#### 【业务规则】

##### 标题
宰后检验记录表

##### 筛选条件

|序号|字段名称|数据类型|必填|默认值|说明/交互规范|
|:--|:--|:--|:--|:--|:--|
|1|**屠宰日期**|日期选择器|是|当前系统日期|必选。  <br>选择具体某一天（YYYY-MM-DD），加载该日宰后检验记录表。|
|2|**审核状态**|下拉选择框|否|全部|**选项如下：**  <br>- 全部<br>- 待审核<br>- 已通过|

##### 记录表结构

页面按所选日期展示 **《宰后检验记录表》**，版式与纸质表单一致。

**表头**

|区域|内容|
|:--|:--|
|主标题|宰后检验记录表（居中）|
|数量单位|数量单位：头|
|表单编号|表单编号：YH/QR-206（右上角）|
|屠宰时间|屠宰时间（年、月、日）：根据所选日期展示|

**表格列**

|列名|说明|
|:--|:--|
|生产线|展示系统生产线名称，如 1号线、2号线、3号线等，每行一条生产线。|
|各生产线屠宰数量|来源于**工控机**当日该生产线屠宰数量，不可手工编辑。|
|胴体、内脏检验|展示“□合格”“□不合格”。  <br>当日该生产线无【宰后检疫不合格】关联记录时勾选“合格”；否则勾选“不合格”。|
|病害畜处理数量|统计所选日期【死猪登记】中类型为**宰后检疫不合格**的记录**条数**，按日汇总，多行生产线纵向合并展示。|
|病害畜产品处理数量（公斤）|汇总所选日期【病害肉登记】记录重量（公斤），多条记录求和，按日汇总，多行生产线纵向合并展示。|
|无害化处理原因及方式|展示病害处理的原因及方式；来源于登记数据摘要，无数据时留空，按日汇总，多行生产线纵向合并展示。|
|总屠宰数|**各生产线屠宰数量之和**，自动计算，不可编辑。|
|合格数量|**总屠宰数 − 病害畜处理数量**，自动计算，不可编辑。|
|检验员|展示所选日期【宰后检验登记（小程序）】的**检验员签名**图片；同一日期、同一生产线一条登记对应一行签名，多条登记多行展示；无登记时留空。按日纵向合并展示。|
|复核人|待审核时留空；已审核时展示复核人签名图片。按日纵向合并展示。|

**数据汇总规则**

|字段|数据来源|计算/展示规则|
|:--|:--|:--|
|各生产线屠宰数量|工控机|按日期 + 生产线取数|
|病害畜处理数量|死猪登记|相同日期，类型 = 宰后检疫不合格，统计记录条数|
|病害畜产品处理数量（公斤）|病害肉登记|相同日期，汇总登记重量（公斤）|
|总屠宰数|系统计算|各生产线屠宰数量之和|
|合格数量|系统计算|总屠宰数 − 病害畜处理数量|
|检验员签名|宰后检验登记（小程序）|相同日期，按生产线匹配展示签名图片|
|胴体、内脏检验|系统判定|当日无宰后检疫不合格死猪登记 → 合格；否则 → 不合格|

**填表说明**（页面底部固定展示）

1. 根据当日屠宰批次顺序登记。
2. 病害畜处理数量为经确认无害化处理的病害猪数量（头）；病害畜产品处理数量为经确认不可食用而无害化处理的动物产品数量（公斤）。
3. “不合格”指胴体、内脏等存在可见病灶、病变淋巴结、寄生虫、气味异常等情形，须按规定无害化处理。
4. 企业应按月（季、年）汇总备查，保存期限不得少于 2 年。

##### 审核信息

对待审核记录，复核人在页面底部操作：

|序号|字段名称|数据类型|必填|不可编辑|默认值|说明/交互规范|
|:--|:--|:--|:--|:--|:--|:--|
|1|**复核人**|文本|是|是|当前登录用户|自动获取当前登录的品控部人员姓名，不可编辑。|
|2|**复核人签名**|签名图片|是|是|【我的】-【签名管理】的签名（如有）|自动获取当前复核人在【签名管理】中的签名图片；无签名时提示先维护签名。  <br>不可手动修改。|

_已审核记录展示复核人、复核人签名及审核时间，不可再次修改。_

##### 按钮

|序号|按钮名称|说明/交互规范|
|:--|:--|:--|
|1|**查询**|位于筛选区，点击后按所选屠宰日期刷新记录表。|
|2|**重置**|位于筛选区，点击后恢复筛选条件为默认值并刷新。|
|3|**导出**|位于筛选区，点击后按当前所选日期导出《宰后检验记录表》Excel，版式与页面一致。  <br>无数据时提示“暂无数据可导出”。|
|4|**通过**|位于页面底部，仅待审核状态可见。  <br>校验复核人签名后，记录复核人、签名及审核时间，将当日记录表置为“已通过”；提示“审核成功”。|

#### 【操作流程】

1. **进入页面**：品控部人员登录 Web 管理系统，进入【屠宰监管】-【宰后检验记录表】，系统默认加载当天的记录表。
2. **选择日期**：选择屠宰日期，点击“查询”加载该日数据；工控机、死猪登记、病害肉登记及小程序登记数据自动汇聚。
3. **核对记录表**：核对各生产线屠宰数量、检验结果、病害处理数量、总屠宰数、合格数量及检验员签名。
4. **复核审核**：确认无误后，系统自动填充复核人及复核人签名，点击“通过”完成当日记录表审核。
5. **导出备查**：需要归档时，选择日期后点击“导出”，下载 Excel 台账。
